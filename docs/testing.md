# 手工验收清单

在 AstrBot 中同时启用 Group Debouncer 与 Heartflow 后，按以下场景验收：

1. 同一人连续发送三条碎片消息：只让最后一条事件继续进入 Heartflow，文本包含三条内容。
2. 两人交错发言：两人的消息各自缓冲，绝不混合或互相取消。
3. `@Bot` 后补充消息：使用 `direct_trigger_window_seconds`，且保留最后消息中的 `At` 组件。
4. `@其他群友`：不得被当作 `@Bot`，应使用普通窗口。
5. `/heartflow`、`/sid` 等命令：立即放行，不进入防抖缓冲。
6. 纯图片消息：立即放行，图片链不被替换。
7. 图文后补充文本：合并后仍保留最后一条消息的图片或其他非文本组件。
8. `max_messages=3`：第三条立即合并并放行，不再等待窗口结束。
9. `reset_timer=true`：在第 0、1、2 秒连续发言时，应约在第 5 秒而不是第 3 秒放行。
10. Heartflow 联动：只处理最终合并事件；此前被停止的旧事件不应进入其主动回复判断。
